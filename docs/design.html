📱 업무 계획관리 앱 — 최종 설계도

이케다 지에 「매일 아침 1시간이 나를 바꾼다」 방법론 기반 · 최종 갱신 2026-07-23

1. 앱의 정체성 (한 문장)

이케다 지에의 '아침 1시간' 방법론을 기반으로, 매일 우선순위를 정하고 하루 일정을 시간순으로 계획하며, 자유롭게 생각을 쏟아내(키보드 또는 펜 손글씨) 새로운 할 일·아이디어를 발굴하고, 매일의 습관을 꾸준히 쌓아가도록 돕는 업무 계획관리 앱.

최종 목표: 외부(시중) 배포 가능한 상용 수준 품질. 우선은 회사 직원 대상. 회사는 태블릿(펜 지원) 보급 환경. B2C 기업이라 일반적으로 회사 프로젝트 업무 중심.

2. 기술 방향

  • 프레임워크: Flutter (Dart) — 안드로이드 우선, 추후 iOS 확장 가능
  • 편집기: VS Code + Flutter 확장 / 빌드·테스트용 Android Studio
  • 개발 PC: Windows 64비트 (개인 PC 또는 회사 팀 노트북에서 작업)
  • 대상 기기: 태블릿 우선(펜 입력 지원). 화면은 폰·태블릿 모두 반응형으로 설계. 테스트 기기: 갤럭시 탭(SM X400) 등
  • 데이터 저장: 1단계는 기기 로컬 저장(Hive) → 추후 로그인·클라우드 동기화 확장
  • 주요 패키지: provider(상태관리), hive·hive_flutter(로컬 저장), shared_preferences(다크모드·성장 동물 설정), intl(날짜 포맷), uuid(고유 ID), table_calendar(월간 캘린더), google_mlkit_digital_ink_recognition(펜 손글씨 인식, ^0.13.0 고정), flutter_local_notifications·timezone(알림), home_widget(홈 화면 위젯, ^0.6.0), speech_to_text(음성 입력), csv·share_plus·path_provider(내보내기), lottie·confetti(연출). 개발용: build_runner, hive_generator.
  • ※ Power Apps와는 코드 호환이 안 됨(별개 결과물). 향후 Teams 활용이 필요하면 Flutter 웹 버전을 Teams 탭으로 삽입하는 경로를 검토.

3. 핵심 사용자 흐름

아침에 앱을 연다 → 오늘의 일정·할 일을 시간순으로 한눈에 본다 → 오늘 할 일을 적는다 → 중요도·긴급도로 분류(2×2 매트릭스) → 오늘 집중할 3가지 선정 → 하루 동안 완료 체크 → 저녁에 회고한다
아침 1시간 루틴: '아침' 탭에서 타이머를 켜고 집중 시간(기본 60분)을 측정 → 끝나면 무엇을 했는지 메모와 함께 기록 저장 → 실천한 날의 연속일수가 쌓임
생각 노트: 부담 없이 쏟아내기(빈 종이) → 키보드 또는 펜 손그림으로 입력 → 나중에 분류 → '할 일로 전환' 버튼으로 실제 태스크와 연결
펜 손그림 → 할 일 흐름(구현 완료): 손그림 노트를 펜으로 작성 → "텍스트로 변환" 버튼으로 손글씨를 텍스트로 변환(ML Kit, 한국어) → 변환된 텍스트가 노트 내용이 됨 → '할 일로 전환'하면 그 텍스트가 오늘 화면 할 일 제목으로 넘어감. 그림 원본은 생각 노트 안에 그대로 남고, 할 일은 텍스트로만 관리(Task에 그림 필드를 두지 않음).
습관·챌린지: 반복하고 싶은 습관을 등록 → 매일 '오늘 했음' 체크 → 연속일수(🔥)가 쌓이며 꾸준함을 시각적으로 확인 → 주간 목표·달력 뷰·통계로 심화 관리
전체 일정 보기: 오늘 화면 헤더의 달력 버튼 → 월간 캘린더에서 날짜별 할 일 개수를 점으로 확인 → 날짜를 골라 그날 일정으로 바로 이동
성장·동기부여: 완료한 할 일·습관 체크·아침 세션이 점수로 쌓임 → 통계 대시보드에서 예스코 마스코트가 알→아기→성체→다 큰→마스터로 성장 → 요약·7일 그래프·카테고리별 완료로 성취를 한눈에 확인
프로젝트 관리: 오늘 화면 우측 상단 폴더 버튼 → 프로젝트를 만들고(이름·색상·기간) 할 일에 프로젝트를 연결 → 프로젝트별 진행률(속한 할 일 완료 비율)을 카드로 확인 → 타임라인(간트) 뷰에서 기간 막대·진행률·오늘 위치를 한눈에 확인(확대/축소 지원)
홈 화면 위젯: 안드로이드 홈 화면에 위젯을 배치하면 오늘의 핵심 할 일(중요+긴급 또는 Top3 우선) 최대 3개가 표시됨 → 할 일 추가/완료/수정/삭제 및 앱 시작 시 실시간 갱신.

5. 화면 구성 (현재까지 구현)

  • 오늘 화면(홈): 오늘 일정을 시간순 리스트로 표시. 상단 '오늘 집중 3가지' 강조, 날짜 앞뒤 이동. 장소·카테고리 필터, 할 일에 장소 아이콘·카테고리 뱃지. 헤더 우측: 달력·프로젝트(폴더)·통계·검색·다크모드 버튼. 날짜 넘김 화살표는 가운데 날짜 양옆 배치.
  • 프로젝트 목록 화면: 색상 점+이름+완료/전체+진행률 바+% 카드. + 버튼으로 추가, 길게 눌러 완료/수정/삭제. 헤더 타임라인 버튼으로 간트 진입.
  • 프로젝트 타임라인(간트) 화면: 기간 막대·진행률·오늘 세로선·날짜 눈금. 확대/축소(+/− 버튼, 하루폭 8~48px) 및 줌 레벨에 따른 날짜 라벨 간격 자동 조정, 막대 탭으로 기간 편집.
  • 통계 대시보드 화면: 성장 마스코트 카드 · 요약 4개 · 최근 7일 완료 막대 · 최근 7일 아침 집중 막대 · 이번 주 카테고리별 완료.
  • 아침 1시간 타이머 화면: 원형 진행 링(기본 60분), 시작/일시정지/리셋, 완료 시 메모 저장, 누적·연속·히스토리.
  • 우선순위 매트릭스: 선택 날짜 할 일을 2×2 사분면 배치(필터 함께 적용).
  • 습관·챌린지 / 습관 상세: 카드(이모지+이름+🔥연속+오늘 체크+주간 진행률), 상세는 월간 달력 체크 뷰+통계.
  • 월간 캘린더 화면: table_calendar 기반, 날짜별 할 일 개수 점 표시, 날짜 탭 미리보기, 해당 날짜로 이동.
  • 할 일·일정 추가·편집: 제목·메모·시간·날짜, 중요도·긴급도, 장소, 카테고리, 프로젝트 선택, 5W2H 접이식, 반복 설정. 음성 입력(마이크 버튼, speech_to_text)으로 제목 받아쓰기 지원.
  • 생각 노트: 빈 종이 방식. 상단 입력창 + 텍스트 추가 버튼 + 손그림 노트 진입 버튼(연필 아이콘). 카드에 손그림이 있으면 미리보기(bounding box 자동 축소·중앙정렬) + "탭하여 수정" 표시. 카드 액션: 할 일로 전환 / 아이디어 보관 / 삭제.
  • 손그림 노트 화면(PenNoteScreen): 상단 선택 메모 입력 + 전체화면 펜 캔버스 + "텍스트로 변환" 버튼(ML Kit). 신규 작성/기존 편집(note 전달 시 기존 획·텍스트 복원) 모두 지원. 저장 시 신규는 addNote, 편집은 updateNotePen 호출.
  • 펜 캔버스 위젯(PenCanvas): 색상 팔레트(검·빨·파·초·주)·굵기 슬라이더(1~12)·지우개·되돌리기·전체지우기. 부모가 준 높이를 Expanded로 채움. 획이 바뀔 때 onChanged로 전달, initialStrokes로 이어 그리기. StrokePreviewPainter로 읽기전용 미리보기 제공.
  • 홈 화면 위젯(네이티브): HomeScreenWidgetProvider(Kotlin) + widget_layout.xml + widget_info.xml. Dart WidgetService가 오늘의 핵심 할 일 3개를 'widget_task_list' 키로 전달·갱신.
  • 저녁 회고 / 검색 / 다크모드: 기존대로.
하단 탭(6개): 오늘 → 아침 → 매트릭스 → 🔥습관 → 생각노트 → 회고. 통계·프로젝트는 오늘 화면 헤더 버튼, 간트는 프로젝트 화면 헤더 버튼으로 진입.

7. 확정된 설계 원칙

  • MVP는 "작동하는 가장 작은 완성품". 다만 난이도가 비슷한 기능은 함께 만든다.
  • 코드 바깥이 얽히는 기능(로그인·클라우드)은 분리해 뒤 단계로. 계정·설정·결정이 필요하기 때문. (알림·펜 변환·위젯은 3단계에서 완료)
  • 생각 노트는 유도 질문 없이 빈 종이 방식.
  • 손그림 노트 → 할 일 전환은 그림이 아니라 "변환된 텍스트"로 넘긴다(확정). Task에는 그림 필드를 두지 않는다. ML Kit 텍스트 변환이 구현되어 손글씨 노트를 변환 후 전환하는 흐름이 완성됨.
  • 펜 그림은 획 좌표(JSON)로 저장해 재편집·색/굵기 유지·손글씨 인식 재사용을 가능하게 한다. Note.penStrokes(HiveField 7)에 문자열로 보관.
  • PenCanvas는 부모가 준 높이를 Expanded로 채운다(height에 double.infinity를 넘기지 말 것 — 레이아웃/히트테스트 오류 원인). 미리보기는 StrokePreviewPainter가 bounding box로 자동 축소·중앙정렬.
  • 화면은 폰과 태블릿 모두 반응형으로 설계.
  • 데이터 접근은 StorageService 한 곳으로 모은다. 노트 저장은 addNote/archiveNote/convertNoteToTask/updateNotePen 모두 storage.saveNote(note)로 통일(개별 note.save() 지양).
  • Hive 모델에 필드/모델을 추가할 때는 로컬에서 dart run build_runner build --delete-conflicting-outputs로 .g.dart를 재생성한다. "0 outputs"면 dart run build_runner clean 후 재빌드. (GitHub 웹에서 .g.dart를 손으로 고치지 말 것)
  • typeId는 겹치지 않게. 현재: Task0/Note1/NoteStatus2/Reflection3/Habit4/MorningSession5/Project6. 다음은 7부터. 필드만 추가면 HiveField 번호만 늘리고 typeId는 유지.
  • 새 typeId 추가 후 기존 설치 기기는 flutter clean 후 재빌드하거나 앱 재설치.
  • GitHub 웹에서 수정한 경우 로컬 빌드 전 git pull origin main. 병합이 꼬이면 git fetch origin 후 git reset --hard origin/main(로컬 변경분 사라짐 주의).
  • Color.value는 deprecated → .toARGB32() 사용. withOpacity도 deprecated → .withValues() 권장(info 수준, 빌드는 통과).
  • [빌드 주의] flutter_local_notifications는 core library desugaring 필수. android/app/build.gradle.kts의 compileOptions에 isCoreLibraryDesugaringEnabled = true 추가 + dependencies 블록에 coreLibraryDesugaring("com.android.tools:desugar_jdk_libs:2.1.4") 추가. (2026-07-23 적용)
  • [빌드 주의] home_widget은 0.5.0+ 필요. 0.3.1은 최신 Flutter의 ViewConfiguration 변경(No named parameter 'size')과 충돌해 웹/데스크톱/안드로이드 빌드 실패. 현재 ^0.6.0 사용. updateWidget 호출 시 androidName 파라미터 함께 지정.
  • [빌드 주의] ML Kit 버전 고정. google_mlkit_digital_ink_recognition는 any 대신 ^0.13.0으로 고정(MissingPluginException 방지). 첫 사용 시 한국어 모델을 인터넷에서 다운로드.
  • [빌드 주의] 16KB ELF 정렬 경고. 최신 안드로이드에서 ML Kit·Flutter 등 서드파티 .so가 16KB 페이지 정렬을 안 따른다는 호환성 경고가 뜰 수 있음. 개발/테스트 단계에서는 계속 진행으로 무시 가능. 플레이스토어 정식 배포 시 각 플러그인 최신 버전·NDK 업데이트로 대응.
  • Windows에서 플러그인 빌드 시 개발자 모드(Developer Mode) 활성화 필요(symlink 지원). 없으면 flutter pub get에서 symlink 경고와 exit code 1.
  • Gradle/AGP/Kotlin 버전 경고, "packages have newer versions"는 빌드를 막지 않는 경고이므로 당장은 무시 가능. 여유 있을 때 일괄 업그레이드 검토.
  • git 사용자 정보: git config --global user.name "lemon6243" / git config --global user.email "93980574+lemon6243@users.noreply.github.com".
  • 시간 측정은 시작 시각을 저장해 DateTime.now()와의 차이로 계산(Timer.periodic은 UI 갱신용만).
  • 에셋은 assets/ 아래(예: assets/growth/)에 두고 pubspec.yaml에 등록. 설계 문서는 docs/(빌드 미포함).
