📱 업무 계획관리 앱 — 최종 설계도

이케다 지에 「매일 아침 1시간이 나를 바꾼다」 방법론 기반 · 최종 갱신 2026-07-16

1. 앱의 정체성 (한 문장)

이케다 지에의 '아침 1시간' 방법론을 기반으로, 매일 우선순위를 정하고 하루 일정을 시간순으로 계획하며, 자유롭게 생각을 쏟아내 새로운 할 일·아이디어를 발굴하고, 매일의 습관을 꾸준히 쌓아가도록 돕는 업무 계획관리 앱.

최종 목표: 외부(시중) 배포 가능한 상용 수준 품질. 우선은 회사 직원 대상. 회사는 태블릿(펜 지원) 보급 환경.

2. 기술 방향

  • 프레임워크: Flutter (Dart) — 안드로이드 우선, 추후 iOS 확장 가능
  • 편집기: VS Code + Flutter 확장 / 빌드·테스트용 Android Studio
  • 개발 PC: Windows 64비트 (개인 PC 또는 회사 팀 노트북에서 작업)
  • 대상 기기: 태블릿 우선. 화면은 폰·태블릿 모두 반응형으로 설계. 펜 입력 고려
  • 데이터 저장: 1단계는 기기 로컬 저장(Hive) → 추후 로그인·클라우드 동기화 확장
  • 주요 패키지: provider(상태관리), hive·hive_flutter(로컬 저장), shared_preferences(다크모드 설정), intl(날짜 포맷), uuid(고유 ID), table_calendar(월간 캘린더). 개발용: build_runner, hive_generator.
  • ※ Power Apps와는 코드 호환이 안 됨(별개 결과물). 향후 Teams 활용이 필요하면 Flutter 웹 버전을 Teams 탭으로 삽입하는 경로를 검토.

3. 핵심 사용자 흐름

아침에 앱을 연다 → 오늘의 일정·할 일을 시간순으로 한눈에 본다 → 오늘 할 일을 적는다 → 중요도·긴급도로 분류(2×2 매트릭스) → 오늘 집중할 3가지 선정 → 하루 동안 완료 체크 → 저녁에 회고한다
아침 1시간 루틴: '아침' 탭에서 타이머를 켜고 집중 시간(기본 60분)을 측정 → 끝나면 무엇을 했는지 메모와 함께 기록 저장 → 실천한 날의 연속일수가 쌓임
생각 노트: 부담 없이 쏟아내기(빈 종이) → 나중에 분류 → '할 일로 전환' 버튼으로 실제 태스크와 연결
습관·챌린지: 반복하고 싶은 습관을 등록 → 매일 '오늘 했음' 체크 → 연속일수(🔥)가 쌓이며 꾸준함을 시각적으로 확인 → 주간 목표·달력 뷰·통계로 심화 관리
전체 일정 보기: 오늘 화면 헤더의 달력 버튼 → 월간 캘린더에서 날짜별 할 일 개수를 점으로 확인 → 날짜를 골라 그날 일정으로 바로 이동

7. 확정된 설계 원칙

  • MVP는 "작동하는 가장 작은 완성품". 다만 난이도가 비슷한 기능은 함께 만들어 불필요하게 미루지 않는다.
  • 코드 바깥이 얽히는 기능(알림·펜 변환·로그인·클라우드·대시보드)은 반드시 분리해 뒤 단계로. 이건 AI 성능 문제가 아니라 님의 계정·설정·결정이 필요하기 때문.
  • 생각 노트는 유도 질문 없이 빈 종이 방식 (placeholder 문구 하나만).
  • 펜 손글씨+텍스트 변환은 최종 목표로 확정하되 순서는 뒤로. 지금은 데이터 자리만 마련.
  • 화면은 폰과 태블릿 모두 쓸 수 있도록 반응형으로 설계.
  • Hive 모델에 필드/모델을 추가할 때는 로컬에서 flutter pub run build_runner build --delete-conflicting-outputs를 돌려 .g.dart를 다시 생성한다. (GitHub 웹에서 .g.dart를 손으로 고치는 건 오류가 잘 나므로 피한다)
  • 새 Hive 모델을 만들 때는 typeId가 겹치지 않게 한다. 현재 사용 중: Task=0, Note=1, NoteStatus=2, Reflection=3, Habit=4, MorningSession=5. 다음은 6부터.
